Accessing Database/Kafka from Spark

Posted on 2018-02-22 | In Spark , Kafka

Problem

As suggested in Design Patterns for using foreachRDD, when you access external systems from withn RDD handler, cautions must be taken to avoid the common mistakes, ie. you should create connections to external systems on Spark Executors.

And, in order to minimize the cost in connection creation and improve system performance, the common technique for DB connection management in Spark applications is using Connection Pool.

However, when it comes to Kafka Consumer/Producer, things become different:

  1. Kafka Producer creation is much more time-consuming, as it needs to discover all partition leader.
  2. Kafka producer is thread-safe.

Solution

A better approach for Kafka access in Spark is taking advantage of Spark Broadcast Variables and Scala lazy val.

For details, refert to Spark and Kafka integration patterns.
