Kafka-Connect

Posted on 2018-04-02 | In Kafka , AWS S3

I got a requirement to load log data from AWS S3 to Kafka, which is not a common use case.
I tried Confluent Kafka Platform with a 3rd-party connector, following the quick-start tutorial on my Mac.
It’s not a pleasant experience.

The 3rd-party connector doesn’t support AWS S3 well, and seems not quite reliable. For example,

  1. Insufficient config entries
    I wasted a lot of time to find why the connector cannot access AWS S3 in China region. I know it’s about the AWS ID and region settings, and tried all expoprts I can find. Still not working…
    It turns out the export AWS_REGION=cn-north-1 doesn’t work. And as the connector doesn’t give the conf entries to specify the AWS S3 endpoint, I have to modify the source code to configure it, like

  2. OOM
    After the connector is finally able to access AWS S3 China region, it can process only a small file set and load it to Kafka, but will run into OOM if the file size gets a little bit larger, like 500MB.
    I tuned the JVM options; it improved a little bit, but finally still failed.

I don’t want to be stuck on this.
Maybe I will create a simple Spark application to read AWS S3 and push the log to Kafka. Spark is cool.
